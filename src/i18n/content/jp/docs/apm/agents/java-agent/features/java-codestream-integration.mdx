コード レベルのメトリクスにより、 [New Relic CodeStream 拡張機能](/docs/codestream/how-use-codestream/performance-monitoring#code-level) を使用する開発者は、コード内の個々のメソッドと一緒に、IDE でコンテキストに応じて表示される APM データを確認できます。 これにより、開発者は、コードを記述およびレビューする際にパフォーマンスの問題に積極的に対処できます。

## はじめに

まず、サポートされているIDEにNew Relic CodeStream拡張機能を[インストール](/docs/codestream/start-here/install-codestream)して、ログインします。

<Callout variant="important">
  コード レベルのメトリックは、Java エージェント バージョン 7.10.0 以降で使用でき、バージョン 7.11.0 以降ではデフォルトで有効になっています。有効または無効にする方法については [、Java エージェント構成: 構成ファイルを](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file/#code-level-metrics) 参照してください。
</Callout>

## エージェントの属性

Java エージェントは、次の属性を報告し、アプリケーションによって生成されたスパンに添付します。

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        **名前**
      </th>

      <th>
        **説明**
      </th>

      <th>
        **例**
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `code.function`
      </td>

      <td>
        インストルメント化された関数 (Java メソッド) の名前。
      </td>

      <td>
        データの取得
      </td>
    </tr>

    <tr>
      <td>
        `code.namespace`
      </td>

      <td>
        `code.function`が定義されている名前空間 (完全修飾クラス名)
      </td>

      <td>
        com.example.MyService
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="important">
  API メソッド`NewRelic.getAgent().getTransaction().startSegment()`を使用して作成されたスパンには、これらの属性は含まれません。これらの属性を含むスパンを作成するには、 `@Trace`でメソッドに注釈を付けるか、XML インストルメンテーションを使用します。
</Callout>
