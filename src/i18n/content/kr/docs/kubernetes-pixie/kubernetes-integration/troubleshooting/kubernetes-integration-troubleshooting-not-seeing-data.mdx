## 문제

New Relic의 Kubernetes 통합을 위한 [설치 절차](/docs/kubernetes-monitoring-integration#install) 를 완료했지만 New Relic 계정에 Kubernetes 데이터가 표시되지 않습니다.

## 해결책

[인프라 에이전트를 배포하고](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent/#install) [Kubernetes 설치 절차](/docs/kubernetes-monitoring-integration#install) 를 완료했지만 Kubernetes 대시보드에 정보가 표시되지 않는 경우 다음 단계를 따르세요.

1. `CLUSTER_NAME` 및 `NRI_LICENSE_KEY` 을(를) 올바르게 구성했는지 확인하십시오.자세한 내용은 [설치 절차](/docs/integrations/host-integrations/host-integrations-list/kubernetes-monitoring-integration#install) 를 참조하십시오.

2. 다음을 실행하여 통합 배포가 성공했는지 확인합니다.

   ```
   kubectl get pods -o wide -n <NEWRELIC_NAMESPACE>
   kubectl get deployments -o wide -n <NEWRELIC_NAMESPACE>
   kubectl get daemonsets -o wide -n <NEWRELIC_NAMESPACE>
   ```

3. [인프라 에이전트 및 통합에서 로그를 검색하고](/docs/integrations/kubernetes-integration/troubleshooting/get-logs-version) [오류 메시지](/docs/integrations/host-integrations/troubleshooting/kubernetes-integration-troubleshooting-error-messages) 가 없는지 확인합니다.
